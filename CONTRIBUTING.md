# Contributing to LFA Reader

Thanks for your interest in contributing.

## Before You Start

- Open an issue before any large feature, architectural change, or workflow change.
- Small bug fixes, documentation fixes, and focused UI polish can usually go straight to a pull request.
- Do not include secrets, `.env` files, databases, uploaded images, or other user data in commits, issues, or pull requests.

## Development Scope

- Keep changes small and targeted.
- Preserve the current single-image workflow unless a maintainer asks for a broader redesign.
- Update documentation when behavior, setup, or operational steps change.

## Pull Requests

- Describe the problem and the exact change you made.
- Include verification details, such as builds, tests, or manual checks.
- Keep pull requests focused. Unrelated refactors are likely to be rejected.

## Licensing

By submitting a contribution, you represent that you have the right to submit it.
Accepted contributions will be made available under the repository license, PolyForm Noncommercial License 1.0.0.
If you cannot agree to that licensing model, do not submit a contribution.

## Maintainer Review

Maintainers may request revisions, decline contributions that do not fit the product direction, or close inactive pull requests.
